# Engineering & Contribution Guide (Kami Excel Extractor)

本プロジェクトにおける開発およびメンテナンスの基準を定義します。コントリビューターは以下のガイドラインに従ってください。

## 1. 非同期処理 (asyncio) の徹底

I/O 待ちが発生する処理は原則として非同期で実装します。

- **非同期優先:** LLM API 呼び出し、画像処理、ファイル操作の重いものは `async` / `await` で実装してください。
- **並列制御:** 外部 API へのリクエストは `asyncio.Semaphore` を使用し、レート制限（`GEMINI_RPM_LIMIT`）を超過しないよう設計してください。
- **ブロッキング回避:** Base64 エンコードや重い I/O 処理は `asyncio.to_thread` を使用してスレッドプールへオフロードし、イベントループのブロッキングを防いでください。
- **二重実行の回避:** ライブラリ内部で `asyncio.run` を呼び出さないでください（`RuntimeError` の原因になります）。同期ラッパーを提供する場合は、クラスの公開インターフェース層でのみ使用してください。

## 2. データの堅牢性と互換性

抽出されたデータが後続のシステム（LLM や JSON 保存）で正しく扱えることを保証します。

- **JSON シリアライズ:** 最終結果には `json.dump` 可能な型（文字列、数値、リスト、辞書）のみを使用してください。`date` や `datetime` は ISO 形式の文字列へ変換してください。
- **画像正規化:** Excel から抽出したメディアデータは、直接送信せず Pillow を介して標準的な PNG 形式（RGB モード）に正規化してください。

## 3. セキュリティ基準

- **一時ファイル管理:** `tempfile.TemporaryDirectory` や `tempfile.NamedTemporaryFile` を使用し、安全なディレクトリ作成と自動削除を徹底してください。
- **パス・サニタイズ:** ユーザー入力（シート名等）をファイルパスに含める場合は、必ず `utils.secure_filename` を通してパス・トラバーサルを防止してください。
- **サブプロセス:** `subprocess.run` を使用する際は、必ず `timeout` 引数を設定し、ゾンビプロセスの発生やリソース占有を防いでください。

## 4. テスト基準

- **コンテナ実行:** LibreOffice 等の依存関係があるため、テストは Docker コンテナ内での実行を正とします。
- **自動テストの追加:** 新機能やバグ修正の際は、必ず `pytest` によるテストを追加してください。特に非同期メソッドのモックが適切であることを確認してください。
- **セキュリティ検証:** セキュリティに関連する変更（一時ファイル、サブプロセス等）は、`test_security_remediation.py` に準じた検証テストを含めてください。

## 5. ドキュメントの更新

機能の変更や追加を行った場合は、`plan/` ディレクトリ内の該当する技術ドキュメントも併せて更新してください。
