# エージェント開発ガイドライン (AGENTS.md)

このリポジトリでAIエージェントが開発作業を行うための簡潔なルールです。

## 1. 開発環境の管理 (uv / Docker)
- **ローカル開発**: パッケージ管理およびスクリプトの実行には `uv` を使用してください。
  - テストの実行: `uv run pytest`
- **コンテナ環境**: LibreOffice 等の依存パッケージを含む環境を Docker で提供しています。
  - コンテナの起動: `docker compose up -d --build`

## 2. テスト駆動開発 (TDD) の徹底
- 新機能の追加やバグ修正を行う際は、**TDD（テスト駆動開発）**を適用してください。
- 変更を行う前にまずテストコード（`tests/` 内）を作成・修正し、テストが失敗することを確認してから実装コードを修正してください。
- 常にテストスイートを実行し、既存の機能を壊していないことを確認しながら進めてください。
