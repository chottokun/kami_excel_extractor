# LLM Usage Cost Analysis

## 概要
`kami_excel_extractor` における LLM 利用コストの解析結果です。

## コスト構造
本システムでは以下のコスト要因を最適化しています。

1. **LLMバイパス**: `is_simple_table` により、構造が単純な表は LLM 呼び出しを回避 ($0コスト)。
2. **ビジュアルサマリー・キャッシュ**: 同一画像は初回のみ解析し、キャッシュにより再呼び出しコストを削減。
3. **Flash系モデルの採用**: コスト効率の高いモデルのデフォルト採用。

## トークン消費予測 (complex_report.xlsx 例)
| フェーズ | 処理対象 | LLM呼び出し頻度 | 推定コスト |
| :--- | :--- | :--- | :--- |
| データ抽出 | 構造化された表 | 低 (バイパス有効時) | 低 |
| 画像解析 | グラフ・現場写真 | 低 (キャッシュ活用) | 低 |
