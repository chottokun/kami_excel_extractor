# **VLM 統合と適応型抽出戦略**

「表形式」「方眼紙」「報告書」が混在するExcelを正確に構造化するため、LiteLLM を介した 2段階の推論プロセス（Chain of Thought）を実装する。

## **1. 推論プロセス: 2-Stage Reasoning**

VLM（Gemini 2.0 Flash等）に対し、一気に抽出させるのではなく、まずレイアウトを理解させるプロセスを踏ませる。

### **Stage 1: レイアウト・セマンティクス分析**
- **問い:** 「この画像はどのような構造か？（表、方眼紙、またはその混合？）」
- **目的:** データの「塊」がどこにあるか、ラベルと値の対応ルール（例：ラベルの右に値がある、あるいは下の結合セルに値がある）を特定する。

### **Stage 2: 座標ベースの精密抽出**
- **ルール:** **「画像で位置を確認し、XMLマップから値を取得せよ」**
- **目的:** 画像の読み取りミス（OCRエラー）を防ぎ、XMLにある「計算式の結果」や「正確な数値」を100%の精度で出力する。

## **2. 汎用抽出プロンプトの構成**

### **システムプロンプト**
> あなたは、日本の特殊なExcel形式（方眼紙、報告書、複雑な表）を解析するエキスパートです。
> 提供される「PNG画像」で視覚的なレイアウト（項目同士の空間的な関係）を把握し、添付の「XMLマップ（JSON）」から正確なテキスト値を引用して、指定されたJSONスキーマに従って出力してください。
> **画像内でのOCRは行わず、必ずXMLマップ内の座標に紐づく値を使用してください。**

### **ユーザーメッセージ（指示例）**
> 1. まず、画像全体を俯瞰し、どのような情報のまとまりがあるか（例：基本情報セクション、明細テーブル、特記事項など）を報告してください。
> 2. 次に、各項目をXMLマップの座標と照らし合わせながら抽出してください。
> 例：「氏名」ラベルが B2 にある場合、その右側の結合セル D2:F2 の値を『氏名』として抽出する。

## **3. LiteLLM 経由のマルチモデル活用**

- **Gemini 2.0 Flash:** デフォルトの高速・高精度なマルチモーダル抽出エンジン。
- **Azure GPT-4o:** より複雑な報告書の文脈理解が必要な場合のフォールバック。

```python
# Pydanticスキーマは抽出対象の業務ドメインに合わせて定義
class NormalizedOutput(BaseModel):
    category: str = Field(description="情報の分類（基本情報、明細など）")
    data_points: List[Dict[str, Any]] = Field(description="ラベルと値のペア。値はXMLマップから引用。")
```

## **4. 精度向上のための「座標ガイド」**

プロンプトに以下のヒントを動的に注入する。
- 「背景色が \#FFFF00 のセルは必須入力項目です。」
- 「太い外枠で囲まれた範囲が一つのデータテーブルです。」
- 「結合されたセルは、一つの長い文章、または大きなヘッダーを意味します。」
