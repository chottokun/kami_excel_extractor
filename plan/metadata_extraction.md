# **構造メタデータとメディアの統合抽出**

Excelドキュメントから「情報の地図（論理構造）」と「物理メディア（埋め込み画像）」を抽出し、VLMによる空間的・文脈的理解を最大化する戦略を定義する。

## **1. ユニバーサル・コーディネート・マップ (The Logical Map)**

特定のレイアウト（表、方眼紙、報告書）に依存せず、全セルの以下の情報を抽出する。

- **テキストと絶対座標:** 全セルの文字列と A1 形式の座標。
- **結合セル・トポロジー:** 結合範囲（例：`B2:F3`）を一つの論理的な「情報ブロック」として定義。
- **視覚的ヒント:** 罫線の有無（テーブルの境界特定）、背景色（重要項目の特定）。

### **抽出の最適化 (`iter_rows`)**
- **高速化:** `ws.cell()` による個別アクセスを廃止し、`ws.iter_rows()` によるバッチ取得を採用。これにより、巨大なシートにおける抽出速度が約4.6倍向上した。
- **データ型変換:** 抽出時に `date` や `datetime` オブジェクトを検知し、自動的に ISO 8601 形式の文字列へ変換。JSON シリアライズ時のエラーを未然に防ぐ。

## **2. メディア（画像）の自動抽出ロジック**

Excel内に埋め込まれた現場写真や図解を、単なる「画像の一部」としてではなく、独立した「データ」として扱う。

- **物理抽出:** `openpyxl` の `_images` プロパティを利用し、`media/` ディレクトリに保存。
- **画像正規化 (Pillow):** 抽出されたバイナリデータは、Pillow を用いて `RGB` モードの PNG 形式に正規化される。これにより、EMF/WMF などの非標準形式や、VLM API でエラーの原因となる特殊なカラープロファイルを排除する。
- **空間マッピング:** 画像が配置されているアンカー座標（例：`A10`）を特定し、論理マップに記録。
- **VLMへの提示:** プロンプト内で「座標 A10 にある写真は現場写真 No.1 です」といった指示を出し、視覚情報と論理情報の不一致（OCRミス）を排除。

## **3. 安全なファイル管理と複数シートの統合**

- **パス安全性の確保:** シート名や画像ファイル名には `secure_filename`（`utils.py`）を適用。日本語文字を維持しつつ、ディレクトリトラバーサル攻撃（`../` 等）を構造的に防ぐ。
- **論理統合:** ワークブック内の全シートを走査し、シート名をキーとした階層的なJSONを生成する。これにより、複数のシートにまたがる複雑な報告書を一つのコンテキストで解析可能にする。

## **4. 物理抽出（Simple Table）モード**

LLMのトークン節約と精度向上のため、以下の要件を満たすシートを「単純な表」として自動検知する。
- 結合セルが存在しない。
- 2行以上のデータがあり、1列以上の構成である。
- 1行目に2つ以上の非空ヘッダーが存在する。

検知された場合、LLMを介さずに Python ロジックのみで構造化リストを生成し、パイプラインの高速化と低コスト化を実現する。
