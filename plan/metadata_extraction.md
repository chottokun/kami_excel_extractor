# **構造メタデータとメディアの統合抽出**

Excelドキュメントから「情報の地図（論理構造）」と「物理メディア（埋め込み画像）」を抽出し、VLMによる空間的・文脈的理解を最大化する戦略を定義する。

## **1. ユニバーサル・コーディネート・マップ (The Logical Map)**

特定のレイアウト（表、方眼紙、報告書）に依存せず、全セルの以下の情報を抽出する。

- **テキストと絶対座標:** 全セルの文字列と A1 形式の座標。
- **結合セル・トポロジー:** 結合範囲（例：`B2:F3`）を一つの論理的な「情報ブロック」として定義。
- **視覚的ヒント:** 罫線の有無（テーブルの境界特定）、背景色（重要項目の特定）。

## **2. メディア（画像）の自動抽出ロジック**

Excel内に埋め込まれた現場写真や図解を、単なる「画像の一部」としてではなく、独立した「データ」として扱う。

- **物理抽出:** `openpyxl` の `_images` プロパティを利用し、PNG等の形式で `media/` ディレクトリに保存。
- **空間マッピング:** 画像が配置されているアンカー座標（例：`A10`）を特定し、論理マップに記録。
- **VLMへの提示:** プロンプト内で「座標 A10 にある写真は現場写真 No.1 です」といった指示を出し、視覚情報と論理情報の不一致（OCRミス）を排除。

## **3. 複数シートの論理統合**

ワークブック内の全シートを走査し、シート名をキーとした階層的なJSONを生成する。これにより、複数のシートにまたがる複雑な報告書（例：概要シート ＋ 写真詳細シート）を一つのコンテキストで解析可能にする。

## **4. データ圧縮 (SheetCompressor 概念の活用)**

情報の密度を保ちつつ、VLMのコンテキスト（トークン）を節約するため、以下の処理を行う。
- **空セルのスキップ:** 値がなく、かつスタイル情報（背景色・罫線）もないセルはマップから除外。
- **情報の集約:** 結合セル内の冗長な座標情報を整理し、トップ座標のみに情報を集約。
