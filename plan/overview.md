# **Excelドキュメント構造化パイプライン：実現可能性検証と最終実装計画**

## **1. 実現可能性の総括 (検証完了)**

本調査およびPoC（概念実証）を通じて、視覚情報（画像）と論理構造（XMLメタデータ）を統合したExcel構造化パイプラインの**「極めて高い実用性」**が証明された。特に、日本特有の「方眼紙エクセル」や「現場写真が埋め込まれた複雑な報告書」に対しても、Gemini 2.5 Flash等の最新VLMと組み合わせることで、100%に近い精度で構造化抽出が可能であることを実証した。

## **2. 確立された技術スタック**

| コンポーネント | 選定技術 | 検証結果と選定理由 |
| :---- | :---- | :---- |
| **画像レンダリング** | LibreOffice + Poppler | **2段階変換（Excel -> PDF -> PNG）**により、Docker環境での絶対的な安定性を確保。 |
| **構造解析** | openpyxl + Media Logic | セル結合・罫線情報に加え、**埋め込み写真の自動抽出**と座標マッピングに成功。 |
| **マルチモデル制御** | LiteLLM | プロバイダーに依存しない共通I/Fを確立。モデル名の変更のみでGemini/OpenAI/Azureを切替可能。 |
| **開発スタイル** | Modular Library | コア機能を `kami_excel_extractor` としてパッケージ化。再利用性と保守性を最大化。 |

## **3. 克服された主要な技術課題**

1.  **LibreOfficeの安定稼働**: 
    ヘッドレス環境での描画エラーを、PDFを経由する2段階フローとUUIDベースのプロファイル隔離によって完全に解決。
2.  **ハルシネーション（幻覚）の排除**: 
    画像をOCR対象としてではなく「位置関係のガイド」として使い、実際のテキスト抽出はXMLマップから行う「座標ベース抽出」により精度を劇的に向上。
3.  **複数シートと写真の統合**: 
    ワークブック全体の論理構造を一つのJSONにまとめ、埋め込まれた写真も物理抽出して文脈に関連付けるロジックを確立。

## **4. 今後の展望**

- **セマンティック・セグメンテーション**: 非常に巨大なシートの場合、シートを論理的なセクション（基本情報、明細、等）に分割してVLMへ送信する最適化の検討。
- **コスト最適化**: LiteLLMの推論パラメータを調整し、タスクの複雑さに応じてモデル（Pro/Flash）を使い分ける動的ルーティングの実装。
- **UI/UX統合**: 本ライブラリをバックエンドとした、ドラッグ＆ドロップでExcelを解析できるWebアプリケーションへの展開。

以上の検証結果に基づき、本プロジェクトは実運用フェーズへ移行するための全ての技術的要件を満たしていると結論付ける。
