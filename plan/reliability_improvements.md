# 構造化データ抽出の信頼性向上計画（JSON移行とリトライロジック）

本計画は、ローカルLLMを用いた際の YAML パースエラーを削減し、システム全体の堅牢性を高めることを目的としています。

## 現象と課題
- `qwen3.5:4b` や `gemma4:latest` などの小型モデルにおいて、YAML の構文エラー（未引用のコロンや括弧の不一致等）が発生し、パースに失敗することがある。
- 現在はパースに失敗した場合、エラー内容をログに残して Markdown に記載するのみとなっており、データ抽出としては未完の状態で停止している。

## 改修内容

### 1. YAML から JSON への移行
ローカルLLMの「JSON Mode」を最大限活用できるよう、出力形式を JSON に変更します。
- プロンプトの要求形式を JSON に更新。
- `litellm.acompletion` において `response_format={"type": "json_object"}` を指定。
- `PyYAML` によるパースから、Python 標準の `json` モジュール（または Pydantic による自動パース）へ切り替え。

### 2. セルフリトライ機能の導入
パースエラーを検知した際、自動的にエラー内容をプロンプトに含めてLLMに再試行を依頼します。
- 初回試行：通常の抽出プロンプト。
- リトライ試行（1回）：エラーメッセージ（どの行でパースエラーが出たか等）を含めた「修正依頼」プロンプト。

### 3. Pydantic による構造定義とバリデーション
LLMからのレスポンスが規定の構造を維持しているか、実行時に検証します。
- `SheetData`, `RagChunk` 等の Pydantic モデルを定義。
- バリデーションエラー時もリトライロジックをトリガー。

## 期待される効果
- 構造化抽出の成功率が大幅に向上。
- JSON Mode による構文レベルのエラー（引用符の閉じ忘れ等）の撲滅。

## 実装済みの信頼性・セキュリティ向上

### 1. 変換エンジンの堅牢化 (3-Layer Fallback Chain)
Excelから画像への変換信頼性を高めるため、PDF経由の変換において以下の多層的なフォールバックを実装。
- **Primary**: `pdftocairo` による高速・高品質なPNG生成。
- **Secondary**: `PyMuPDF (fitz)` によるPythonネイティブなレンダリング。
- **Tertiary**: `ImageMagick` による汎用的な変換。

### 2. セキュリティの徹底 (Security-by-Design)
- **CWE-426 対策**: `shutil.which` を使用し、外部コマンド（soffice, pdftocairo, magick等）の実行ファイルを絶対パスで解決。
- **引数注入対策**: 外部コマンド実行時のパス引数を常に絶対パス化。
- **パス・トラバーサル防止**: `DocumentGenerator` において `secure_filename` を導入し、出力ファイル名のサニタイズを徹底。
- **XSS 対策**: レポート生成時に全外部入力を HTML エスケープし、不正なスクリプト混入を防止。

### 3. 画像抽出のエラーハンドリング
- **個別抽出の継続**: シート内の特定の画像が破損または未対応形式（EMF/WMF/zlib圧縮図形等）であっても、エラーをキャッチしてログに記録し、他の正常な画像の抽出を継続する設計を導入。
- **フォーマット正規化**: 抽出した画像は Pillow で PNG に変換・正規化し、後続の VLM 解析の成功率を向上。

## 今後の課題

### 1. 特殊な埋め込み形式への対応
Excel内の「図形（Shape）」や「メタファイル（EMF/WMF）」、zlib圧縮された内部形式での画像埋め込みは現在抽出対象外。
- **対策案**: `pyvips` や `Wand (ImageMagick)` 等の強力なライブラリを導入し、中間形式を介さず直接解析する抽出エンジンの強化を検討。
