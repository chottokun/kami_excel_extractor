# AGENTS.md

本システムにおける主要なコンポーネントを、自律的な専門エージェントとして定義します。各エージェントは特定のタスクに特化し、`KamiExcelExtractor` (Core Orchestrator) によって統合・制御されます。

## 1. エージェント定義一覧

### 🛡️ Core Orchestrator (`core.py`)
- **役割**: 全プロセスを統括する指揮官。
- **責務**: 
    - 各専門エージェントへのタスク割り当てと非同期制御。
    - LLM (VLM) との対話、RPM制限の遵守。
    - 抽出データの整合性検証と最終シリアライズ。
- **特性**: 高い柔軟性を持ち、設定に応じてモデルや処理フローを動的に変更。

### 🎨 Visual Converter (`converter.py`)
- **役割**: Excelの「見た目」を正確にデジタル画像化する専門家。
- **責務**: 
    - LibreOffice と pdftocairo を用いた PDF 経由の高品質画像変換。
    - プロファイル隔離による安定稼働と、画像パスの管理。
- **特性**: ヘッドレステクノロジーに精通し、方眼紙エクセルのレイアウトを100%再現。

### 🔍 Metadata Extractor (`extractor.py`)
- **役割**: Excelの「内部構造」と「埋め込み資産」を抽出する専門家。
- **責務**: 
    - セル結合、背景色、論理座標を維持した HTML テーブルの生成。
    - 現場写真などのメディアファイルの物理抽出と位置特定。
    - 単純な表構造の自動検知（VLMバイパス判断）。
- **特性**: `openpyxl` を駆使し、非可視領域のデータも見逃さない。

### 📝 RAG Architect (`rag_converter.py`)
- **役割**: 抽出データを「AIが読みやすい形式」に再構築する編集者。
- **責務**: 
    - JSONから Markdown (Table/Key-Value) への構造変換。
    - 検索精度を高めるためのセクション分割（チャンク化）。
    - 視覚情報とテキスト情報のコンテキスト統合。
- **特性**: 埋め込みベクトルの検索効率を最大化するための構造設計。

### 📜 Document Generator (`document_generator.py`)
- **役割**: 抽出結果を「人間が確認しやすい形式」にレンダリングするデザイナー。
- **責務**: 
    - Markdownから HTML/PDF へのセキュアな変換。
    - 視覚的サマリー（画像説明）を含めたビジュアルレポートの生成。
- **特性**: セキュリティ（XSS対策）を重視しつつ、可読性の高いレイアウトを提供。

## 2. エージェント間連携図

1.  **Core Orchestrator** が解析依頼を受理。
2.  **Visual Converter** が Excel を PNG画像 に変換（視覚コンテキストの作成）。
3.  **Metadata Extractor** が HTML構造 と 写真 を抽出（論理構造の解析）。
4.  **Core Orchestrator** が画像と構造を統合し、外部VLMへ推論を依頼。
5.  **RAG Architect** が推論結果を Markdown に変換し、RAG用にチャンク化。
6.  **Document Generator** が最終的な PDF レポートを出力。
