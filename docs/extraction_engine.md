# Kami Excel Extractor 抽出エンジン技術解説

本プロジェクトの抽出エンジンは、従来の「単なるテキスト抽出」を超え、人間がExcelシートを目視して理解するプロセスをデジタルに再現するように設計されています。

## コア・アーキテクチャ

抽出パイプラインは以下の 5 つのレイヤーで構成されています。

### 1. セマンティック・クリーニング (Semantic Cleaning)
神エクセル特有の「見た目を整えるための不自然な空白」を正規表現で自動検知し、単語を復元します。
- **対象**: 漢字、ひらがな、カタカナの間に挟まった 1〜3 つの空白。
- **効果**: LLM が「氏　　名」を「氏名」として正しく認識でき、キーの不一致を劇的に減らします。

### 2. 視覚スタイル抽出 (Style-Aware Extraction)
セルの属性を CSS 形式で HTML テーブルに埋め込みます。
- **罫線 (Borders)**: `thick` や `medium` などの線を検知し、セクションの境界線として LLM に伝えます。
- **背景色 (Colors)**: ヘッダー行や重要項目を色情報から推論可能にします。
- **座標 (Coordinates)**: `data-coord` 属性により、AI がエクセル上の絶対位置を参照して思考できます。

### 3. Visual Intelligence (図表データ統合)
画像やグラフを単なる「添付ファイル」ではなく「データソース」として扱います。
1. **メディア解析**: VLM (Vision Language Model) がグラフ画像を読み取り、Markdown テーブルに変換します。
2. **コンテキスト注入**: 変換されたデータを、HTML テーブル内の「画像が配置されていたセル」の直後に動的に挿入します。
3. **効果**: グラフの中にしかない数値を、AI がテキストとして正確に抽出結果に含めることができます。

### 4. ロジック・インジェクション (Logic-Aware Extraction)
エクセルの「計算式」と「表示形式」を抽出します (オプション)。
- **計算式 (Formulas)**: `=SUM(...)` などの数式を抽出し、AI に「このセルは合計値である」という確信を与えます。
- **表示形式 (Number Formats)**: `JPY` や `PERCENT` などの単位を特定し、数値の意味を明確にします。

### 5. マルチモーダル・オーケストレーション
上記すべての情報を統合し、マルチモーダル LLM (Gemini 1.5 Pro 等) に渡します。AI は「画像コンテキスト」「スタイル付きHTML」「数式メタデータ」の3つを照らし合わせることで、極めて高い精度の構造化 JSON を生成します。

## パイプライン・フロー

1. **基本解析**: `MetadataExtractor` がテキスト、スタイル、数式、画像を抽出。
2. **画像解析**: 各画像の内容を VLM で構造化テキスト化。
3. **データ合成**: 画像由来のデータを HTML の該当座標に注入。
4. **AI 推論**: 強化されたコンテキストを用いて JSON を生成。
5. **整合性チェック**: Pydantic スキーマによる最終バリデーション。
